## B.3 Field Validation Rules

**For ITSM Tool Configuration:**

| Field | Validation Rule |
|-------|----------------|
| Change Title | Mandatory, 10-80 characters |
| Change Description | Mandatory, min 50 characters |
| Business Justification | Mandatory |
| Affected Systems | Mandatory, select from CMDB |
| Priority | Mandatory dropdown |
| Impact Level | Mandatory dropdown |
| Likelihood Level | Mandatory dropdown |
| Implementation Date | Mandatory, future date |
| Rollback Plan | Mandatory for Normal changes |
| Approvals | Mandatory before status change to "Scheduled" |

## B.4 Help Text Examples

**Change Description:**  
*"Provide clear, detailed description of the change. Include: what systems/applications are affected, what will be modified (added/changed/removed), and technical approach."*

**Business Justification:**  
*"Explain WHY this change is needed. Examples: fix critical bug, meet compliance requirement, improve performance, enable new feature, reduce costs."*

**Impact Level:**  
*"Consider scope of impact if change fails. Low=single user/system, Medium=team/multiple systems, High=department/major system, Critical=organization-wide/mission-critical."*
