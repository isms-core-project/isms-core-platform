## 📋 DATA LEAKAGE PREVENTION (DLP) - QUICK REFERENCE

**What is DLP?** Technology that monitors and protects sensitive data from unauthorized disclosure.

---

## ✅ DO's

✓ **Classify your data** - Know what's Public, Internal, Confidential, or Restricted  
✓ **Use approved channels** - Secure email, encrypted USB, corporate cloud (M365, Google Workspace)  
✓ **Request exceptions properly** - Use DLP Exception Request Form for legitimate business needs  
✓ **Report incidents** - If you accidentally send sensitive data, report immediately to SOC  
✓ **Attend training** - Complete annual DLP awareness training  
✓ **Ask questions** - Contact IT Support if you're unsure about data handling  

---

## ❌ DON'Ts

✗ **Don't email Restricted data externally** - Use secure portal or encrypted email  
✗ **Don't use personal cloud** - No Dropbox, personal OneDrive, Google Drive for work data  
✗ **Don't use personal USB** - Use company-issued encrypted USB drives only  
✗ **Don't bypass DLP** - VPN to personal account, encryption to avoid detection = policy violation  
✗ **Don't share credentials** - Never email passwords, API keys, private keys  
✗ **Don't ignore DLP alerts** - If DLP blocks you, there's a reason - request exception if legitimate  

---

## 🗂️ DATA CLASSIFICATION QUICK GUIDE

| Classification | Examples | Can I Email Externally? |
|----------------|----------|-------------------------|
| **Restricted** | PII, credentials, payment cards, IP, source code | ❌ NO (Use secure portal) |
| **Confidential** | Contracts, financial reports, business plans | ⚠️ Only to approved partners with encryption |
| **Internal** | Policies, procedures, org charts | ⚠️ Case-by-case (ask manager) |
| **Public** | Marketing materials, public website content | ✅ YES |

---

## 📧 WHAT CHANNELS ARE MONITORED?

| Channel | Monitored? | What's Blocked? |
|---------|------------|-----------------|
| **Email** | ✅ Yes | Restricted data to external recipients, credentials to anyone |
| **Web/Cloud** | ✅ Yes | Uploads to personal cloud, unapproved SaaS |
| **USB Drives** | ✅ Yes | Restricted data to any USB (use encrypted USB only) |
| **Print** | ✅ Yes (logged) | Restricted data print-to-PDF blocked |
| **Mobile** | ✅ Yes (if MDM-managed) | Personal cloud access, screenshots in corporate apps |
| **Personal Devices** | ❌ No | BYOD not monitored (use corporate devices for sensitive data) |

---

## 🆘 WHAT TO DO IF DLP BLOCKS YOU

**Step 1:** Read the DLP notification message (explains why blocked)  
**Step 2:** Determine if this is legitimate business need or policy violation  
**Step 3:** If legitimate:
- Use approved alternative (secure portal, encrypted USB, corporate cloud)
- OR request exception: [DLP Exception Request Form Link]

**Step 4:** If accidental violation:
- No penalty for honest mistakes
- Contact your manager if you have questions

---

## 🚨 HOW TO REPORT AN INCIDENT

**If you accidentally sent sensitive data externally:**

1. **Don't panic** - Honest mistakes happen
2. **Report immediately** - Email: [dlp-incidents@company.com] or Phone: [XXX-XXX-XXXX]
3. **Provide details:**
   - What data was sent?
   - Where did you send it (recipient email, cloud, USB)?
   - When did this happen?
4. **We'll help:** SOC will contain the incident, recover data if possible, prevent recurrence

**Early reporting = faster containment = less damage**

## ⚖️ CONSEQUENCES OF POLICY VIOLATIONS

**Unintentional Violations:**
- First time: Warning + education
- Repeated: Manager notification, enhanced monitoring

**Intentional Violations:**
- Bypassing DLP controls: Suspension or termination
- Data theft: Termination + legal action

**Remember:** DLP monitoring is for security and compliance, not to spy on you. Legitimate business needs are always accommodated via exception process.

---

## 🔐 KEY TAKEAWAYS

1. **Classify data before sharing** - When in doubt, ask your manager
2. **Use approved channels** - Secure portal, corporate cloud, encrypted USB
3. **Request exceptions for legitimate needs** - We want to enable your work, not block it
4. **Report incidents immediately** - Early detection = faster recovery
5. **Complete training annually** - Stay informed on best practices
