# ISMS-IMP-A.5.31-S4: Regulatory Monitoring Process
## SKELETON WITH WRITING INSTRUCTIONS

**Status**: SKELETON - Ready for content development  
**Target Length**: 8-10 pages  
**Purpose**: Step-by-step guide for monitoring regulatory landscape and managing changes

**Related Policy**: ISMS-POL-A.5.31-S4 (Change Management & Evidence Framework)

---

## SECTION STRUCTURE:

1. **Process Overview** (1-2 pages)
   - Purpose, when used, who performs, output
   - Process flowchart: Monitor → Detect → Assess → Update → Communicate
   - Timeline and key artifacts

2. **Step 1 - Establishing Monitoring Infrastructure** (2-3 pages)
   - Setting up monitoring sources (detailed how-to for each source from POL-S4)
   - Configuring alerts and notifications
   - Subscription management
   - Calendar/schedule setup for periodic reviews
   - Example monitoring dashboard setup

3. **Step 2 - Detecting Regulatory Changes** (2 pages)
   - Daily/weekly monitoring routines
   - How to interpret alerts
   - Distinguishing material changes from minor updates
   - Logging changes in Regulatory Change Register
   - Change types: New regulation, Amendment, Repeal, Guidance, Enforcement action

4. **Step 3 - Impact Assessment** (2-3 pages)
   - Detailed walkthrough of impact assessment (6 steps from POL-S4 Section 3.2)
   - Using Impact Assessment Template
   - Decision criteria for each step
   - Example impact assessments
   - When to escalate to management

5. **Step 4 - Updating Framework Documents** (1-2 pages)
   - Which documents to update (POL-00, Requirements Register, Mapping Matrix, etc.)
   - Update procedures
   - Version control
   - Traceability maintenance

6. **Step 5 - Communication & Reporting** (1 page)
   - Notification templates
   - Stakeholder communication
   - Reporting regulatory changes to management

7. **Example Regulatory Changes & Responses** (1-2 pages)
   - 3-4 scenarios showing process in action
   - Generic examples

8. **Templates & Tools** (1 page)
   - Regulatory Change Register template
   - Impact Assessment Template
   - Communication templates

9. **Quality Checks & Common Pitfalls** (1 page)

10. **Related Documents & References** (0.5 page)

**Total: 8-10 pages**

---

## KEY WRITING NOTES FOR IMP-S4:

- **Emphasize ongoing/continuous nature** - this is not one-time
- Provide practical guidance on setting up alerts and monitoring
- Impact assessment is critical - make it systematic
- Show how changes flow through to framework updates
- Examples should show various change types (amendment, new law, repeal, etc.)
- Make it scalable (works for 5 regulations or 50 regulations)
