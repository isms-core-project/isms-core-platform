# Dashboard Overview

## Purpose & Scope

**Dashboard Name:** ISMS-IMP-A.5.32-33.4 - Information Protection Compliance Dashboard

### ISO/IEC 27001:2022 Control References

**Control A.5.32 - Intellectual Property Rights:**

> *The organisation should implement appropriate procedures to protect intellectual property rights.*

**Control A.5.33 - Protection of Records:**

> *Records should be protected from loss, destruction, falsification, unauthorised access and unauthorised release in accordance with legislative, regulatory, contractual and business requirements.*

### What This Dashboard Provides

This dashboard consolidates metrics from all A.5.32-33 assessments to provide:

- **Executive Summary** of overall compliance status
- **Key Performance Indicators (KPIs)** for IP and records protection
- **Control Assessment Results** for A.5.32 and A.5.33
- **Maturity Assessment** using CMMI-based scoring
- **Risk Register** for information protection risks
- **Remediation Tracking** for identified gaps
- **Trend Analysis** showing improvement over time


### Key Principle

This dashboard is an **aggregation and reporting tool** - it pulls data from the foundational assessments (A.5.32-33.1, A.5.32-33.2, A.5.32-33.3) to provide management visibility without duplicating detailed assessment work.

### Dashboard Data Sources

| Data Element | Source Assessment | Update Frequency |
|--------------|-------------------|------------------|
| IP Asset Inventory Metrics | A.5.32-33.1 | Annual |
| IP Protection Status | A.5.32-33.1 | Annual |
| Third-Party IP Compliance | A.5.32-33.1 | Quarterly |
| Software License Compliance | A.5.32-33.1 | Quarterly |
| Records Protection Status | A.5.32-33.2 | Annual |
| Integrity Verification Results | A.5.32-33.2 | Annual |
| Retention Schedule Compliance | A.5.32-33.3 | Quarterly |
| Disposal Completion Rate | A.5.32-33.3 | Quarterly |

# Completion Workflow

## High-Level Process

```
1. GATHER SOURCE DATA (From A.5.32-33.1, .2, .3)
   |
2. UPDATE EXECUTIVE SUMMARY (Overall status)
   |
3. CALCULATE KPIs (Target vs actual)
   |
4. COMPLETE CONTROL ASSESSMENT (A.5.32 and A.5.33)
   |
5. UPDATE MATURITY SCORES (By domain)
   |
6. REFRESH RISK REGISTER (Current risks)
   |
7. UPDATE REMEDIATION STATUS (Gap closure)
   |
8. ANALYSE TRENDS (Period over period)
   |
9. COLLECT EVIDENCE (Dashboard support)
   |
10. REVIEW & APPROVE (Stakeholder sign-off)
```

## Detailed Workflow

### Phase 1: Gather Source Data (30-60 minutes)

**Objective:** Collect metrics from source assessments

**Steps:**
1. Open A.5.32-33.1 (IP Rights Inventory)
   - Extract IP asset count
   - Extract protection status percentages
   - Extract third-party compliance metrics

2. Open A.5.32-33.2 (Records Protection)
   - Extract records category count
   - Extract protection control effectiveness
   - Extract integrity verification results

3. Open A.5.32-33.3 (Retention & Disposal)
   - Extract retention compliance metrics
   - Extract disposal completion rate
   - Extract exception count


### Phase 2: Update Executive Summary (15-30 minutes)

**Objective:** Provide high-level status view

**Complete Sheet 2 (Executive Summary):**
- Overall compliance status (Compliant/Partial/Non-Compliant)
- Key achievements this period
- Key concerns requiring attention
- Assessment period and dates


### Phase 3: Calculate KPIs (30-45 minutes)

**Objective:** Track performance against targets

**Complete Sheet 3 (Compliance Metrics):**

| Metric Category | Sample KPIs | Target |
|-----------------|-------------|--------|
| **IP Protection** | IP assets with designated owners | 100% |
| | IP assets with effective protection | 95% |
| | Third-party IP compliance rate | 100% |
| | Software license compliance rate | 100% |
| **Records Protection** | Records with assigned retention | 100% |
| | Records protection control effectiveness | 90% |
| | Integrity verification pass rate | 100% |
| **Retention/Disposal** | Records disposed on schedule | 95% |
| | Overdue disposals | < 5% |
| | Legal hold compliance | 100% |


### Phase 4: Control Assessment (30-45 minutes)

**Objective:** Evaluate A.5.32 and A.5.33 control effectiveness

**Complete Sheet 4 (Control Assessment):**

For each control requirement:
- Implementation status
- Evidence available
- Gap identified (if any)
- Compliance score (0-5 scale)
- Overall control status


### Phase 5: Maturity Assessment (30-45 minutes)

**Objective:** Score program maturity

**Complete Sheet 5 (Maturity Assessment):**

Use CMMI-based maturity scale:
- 0 - Non-existent: No awareness, no processes
- 1 - Initial: Ad hoc, unpredictable
- 2 - Developing: Reactive, informal
- 3 - Defined: Documented, standardised
- 4 - Managed: Measured and controlled
- 5 - Optimised: Continuous improvement

**Domains to Assess:**
- Policy & Governance
- IP Identification & Classification
- IP Protection Controls
- Third-Party IP Compliance
- Records Classification
- Records Protection
- Retention Management
- Disposal Process
- Monitoring & Metrics


### Phase 6: Risk Register (15-30 minutes)

**Objective:** Document current risks

**Complete Sheet 6 (Risk Register):**
- Risk description
- Risk category (IP/Records/Compliance)
- Likelihood and impact
- Risk score
- Current mitigation
- Owner and status


### Phase 7: Remediation Tracking (15-30 minutes)

**Objective:** Track gap closure

**Complete Sheet 7 (Remediation Tracker):**
- Consolidate gaps from source assessments
- Update progress percentage
- Update status (Open/In Progress/Complete)
- Note any blockers


### Phase 8: Trend Analysis (15-30 minutes)

**Objective:** Show progress over time

**Complete Sheet 8 (Trend Analysis):**
- Compare current metrics to previous periods
- Note improving/stable/declining trends
- Identify areas of concern


### Phase 9: Evidence Collection (15 minutes)

**Complete Sheet 9 (Evidence Register):**
- Link to source assessment evidence
- Document dashboard-specific evidence


### Phase 10: Review & Approval (30 minutes)

**Complete Sheet 10 (Approval & Sign-Off):**
- Schedule review meetings
- Obtain stakeholder sign-offs

# Common Pitfalls

**MISTAKE: Creating metrics without source data**
- All metrics must trace to source assessments
- Don't estimate - measure

**MISTAKE: Not updating regularly**
- Dashboard loses value if stale
- Quarterly updates minimum

**MISTAKE: Reporting only good news**
- Include concerns and gaps
- Executive stakeholders need accurate picture

**MISTAKE: Not tracking trends**
- Point-in-time data is less valuable
- Show progress (or regression) over time

**MISTAKE: Orphaned remediation actions**
- Every gap needs an owner
- Track to closure

**MISTAKE: Not validating source data**
- Check source assessments are current
- Note data currency in dashboard

**MISTAKE: Over-complicating metrics**
- Keep KPIs simple and actionable
- 10-15 metrics is sufficient

**MISTAKE: Not aligning with business priorities**
- Connect metrics to business risk
- Executive audience needs business context

**MISTAKE: Skipping maturity assessment**
- Maturity shows capability, not just compliance
- Track maturity improvement over time

**MISTAKE: Not preparing for audit**
- Dashboard is audit evidence
- Ensure completeness and accuracy
