## Issue Summary

The Excel workbook generators for A.8.24 define Status field dropdown validations with emoji-prefixed values, but the population scripts were inserting plain text values that did not match the validation lists.

## Root Cause

**Generator Validation (Correct):**
```python
formula1='"✅ Compliant,⚠️ Partial,❌ Non-Compliant,N/A"'
```

**Population Script Values (Incorrect):**
```python
["www.company.com", "TLS 1.3", ..., "Compliant", ...]  # Missing emoji prefix
```

## Dashboard External Link Requirements

The Compliance Summary Dashboard (`ISMS-IMP-A.8.24.5`) uses external workbook formulas to pull compliance totals from the four assessment workbooks. These formulas reference the normalized short filenames:

```
='[ISMS-IMP-A.8.24.1.xlsx]Summary Dashboard'!G16
='[ISMS-IMP-A.8.24.2.xlsx]Summary Dashboard'!B11
```

### Important: Cached Values Required

Excel external links read **cached formula values**, not live calculations. Source workbooks generated by Python (openpyxl) contain formulas but no cached values until opened and saved in Excel.

**Before the Dashboard will display data:**

1. Open each source workbook in Excel:
   - `ISMS-IMP-A.8.24.1_Data_Transmission_*.xlsx`
   - `ISMS-IMP-A.8.24.2_Data_Storage_*.xlsx`
   - `ISMS-IMP-A.8.24.3_Authentication_*.xlsx`
   - `ISMS-IMP-A.8.24.4_Key_Management_*.xlsx`

2. **Save each workbook** (Cmd+S / Ctrl+S) — this caches calculated values

3. Re-run the normalization script to copy updated files to short names

4. Open the Dashboard — external links will now resolve correctly
