## Assessment Overview

### Purpose & Scope

**Assessment Name:** ISMS-IMP-A.7.1-2-3-S4 - Physical Access Control Compliance Dashboard

#### What This Dashboard Provides

This dashboard consolidates the three physical access control assessments into a unified executive view:

- **A.7.1 Perimeter Security** (from S1) - Security zone definitions and perimeter construction
- **A.7.2 Entry Control** (from S2) - Access control systems, visitor management, contractor access
- **A.7.3 Secure Areas** (from S3) - Office security, server rooms, meeting rooms

The dashboard provides:

- Overall physical access control compliance score
- Control-by-control compliance status
- Gap analysis with prioritised remediation
- Key performance indicators (KPIs)
- Consolidated evidence register
- Executive approval workflow
- Trend analysis over time
- Risk-based prioritisation

#### ISO 27001:2022 Control Reference

This dashboard covers three related controls:

> *"Security perimeters should be defined and used to protect areas that contain information and other associated assets."* --- Control A.7.1

> *"Secure areas should be protected by appropriate entry controls and access points."* --- Control A.7.2

> *"Physical security for offices, rooms and facilities should be designed and implemented."* --- Control A.7.3

#### Key Principle

This is a CONSOLIDATION dashboard - it brings together completed assessments from S1, S2, and S3. You should NOT complete this dashboard before completing the source assessments.

The dashboard serves as the PRIMARY audit deliverable for physical access control, providing a single view of the organisation's physical security posture.

#### What You'll Document

**Consolidated Compliance:**

- Overall physical access control compliance percentage
- Per-control compliance scores (A.7.1, A.7.2, A.7.3)
- Compliance trends over time
- Comparison to target thresholds

**Gap Analysis:**

- All gaps from S1, S2, S3 assessments
- Severity classification (Critical, High, Medium)
- Impact assessment
- Remediation actions and owners
- Target completion dates
- Progress tracking

**Key Performance Indicators:**

- Perimeter inspection completion rate
- Entry point coverage percentage
- Visitor sign-in compliance rate
- Badge loss/theft incident count
- Access rights review completion
- After-hours access logging rate
- Server room MFA compliance
- Clean desk audit compliance
- Tailgating incident count
- Unauthorised access attempt count

**Consolidated Evidence:**

- Evidence from all three source assessments
- Cross-references to source evidence registers
- Evidence accessibility verification
- Retention compliance

## Dashboard Components

### Executive Dashboard (Sheet 2)

**Purpose:** High-level compliance status and key metrics

**What It Shows:**

- Overall Physical Access Control Compliance Score (weighted average)
- Control-by-control compliance:
  - A.7.1 - Physical Security Perimeters
  - A.7.2 - Physical Entry
  - A.7.3 - Securing Offices, Rooms and Facilities
- Critical, High, and Medium priority gap counts
- Key performance indicators (KPIs)
- Compliance trend indicators (improving/stable/declining)

**Data Source:** Auto-calculated from source assessments (S1, S2, S3)

**Actions Required:**

- Review compliance scores for accuracy
- Verify gap counts
- Prepare executive summary for management review
- Identify areas requiring immediate attention

**Compliance Score Calculation:**

The overall compliance score is calculated as a weighted average:

| Control | Weight | Score Source |
|---------|--------|--------------|
| A.7.1 Perimeter Security | 35% | S1 Summary Dashboard |
| A.7.2 Entry Control | 40% | S2 Summary Dashboard |
| A.7.3 Secure Areas | 25% | S3 Summary Dashboard |

**Threshold Interpretation:**

| Score | Status | Action Required |
|-------|--------|-----------------|
| 90-100% | Compliant | Monitor and maintain |
| 70-89% | Partial | Address high-priority gaps |
| Below 70% | Non-Compliant | Immediate remediation required |

### Gap Analysis (Sheet 3)

**Purpose:** Detailed gap identification and remediation planning

**What to Document (Per Gap):**

**Column A - Gap ID:**

- Unique identifier: "GAP-001", "GAP-002", etc.
- Number sequentially across all controls

**Column B - Control:**

- Source control: "A.7.1", "A.7.2", or "A.7.3"

**Column C - Finding:**

- Brief description of the gap
- Reference source assessment sheet and row

**Column D - Severity:**

- Dropdown: "Critical", "High", "Medium"
- See Severity Classification below

**Column E - Impact:**

- Description of potential impact if gap not addressed

**Column F - Current State:**

- Description of current situation

**Column G - Required State:**

- Description of what compliance looks like

**Column H - Remediation Action:**

- Specific action to close the gap

**Column I - Owner:**

- Named individual (not role) responsible for remediation

**Column J - Target Date:**

- Date by which remediation should be complete

**Column K - Status:**

- Dropdown: "Open", "In Progress", "Closed", "Deferred"

**Column L - Notes:**

- Progress notes, blockers, updates

**Severity Classification:**

| Severity | Definition | Target Remediation |
|----------|------------|-------------------|
| Critical | Immediate risk to critical assets, compliance violation, regulatory exposure | 30 days |
| High | Significant security gap, potential compliance issue | 60 days |
| Medium | Security improvement opportunity, best practice gap | 90 days |

**Actions Required:**

- Extract gaps from S1, S2, S3 assessments (any Non-Compliant or Partial status)
- Prioritise by severity and impact
- Assign remediation owners (named individuals)
- Set realistic target completion dates
- Track progress at least monthly

### KPIs & Metrics (Sheet 4)

**Purpose:** Performance indicators for ongoing monitoring

**Key Metrics:**

| Metric | Measurement | Target | Data Source |
|--------|-------------|--------|-------------|
| Perimeter Inspection Completion | % | 100% | S1 - Sheet 3 |

| After-Hours Access Logging | % | 100% | S2 - Sheet 5 |
| Server Room MFA Compliance | % | 100% | S3 - Sheet 3 |
| Clean Desk Audit Compliance | % | 95%+ | S3 - Sheet 2 |
| Tailgating Incidents | Count/Month | <3 | Security incident reports |
| Unauthorised Access Attempts | Count/Month | 0 | Access control logs |

**What to Document (Per KPI):**

**Column A - KPI:**

- Metric name

**Column B - Measurement:**

- How measured (percentage, count, etc.)

**Column C - Current:**

- Current value from this assessment period

**Column D - Previous:**

- Value from previous assessment period (for trend)

**Column E - Target:**

- Target value from policy

**Column F - Status:**

- Dropdown: "Met", "Near", "Below Target"

**Column G - Trend:**

- Dropdown: "Improving", "Stable", "Declining"

**Column H - Notes:**

- Context, actions planned

**Actions Required:**

- Extract current values from source assessments
- Compare against targets
- Document trends (compare to previous period)
- Identify KPIs requiring intervention

```
   |

   |

   |

```

### Detailed Step-by-Step

**Step 1: Verify Source Assessments (30 minutes)**

- Confirm S1, S2, S3 workbooks are complete
- Confirm all three have received Level 4 (CISO) approval
- Retrieve final versions of all three workbooks
- Note assessment dates and any limitations

**Step 2: Extract Compliance Scores (1 hour)**

- Open Sheet 2 (Executive Dashboard)
- Extract compliance scores from each source assessment's Summary Dashboard
- Calculate weighted overall compliance score
- Extract gap counts by severity
- Review for accuracy

**Step 3: Compile Gap Analysis (1-2 hours)**

- Open Sheet 3 (Gap Analysis)
- Systematically review each source assessment for Non-Compliant and Partial items
- Create gap entry for each identified issue
- Classify severity based on criteria
- Assign remediation owners
- Set target dates
- Ensure no gaps are missed

**Step 4: Calculate KPIs (1 hour)**

- Open Sheet 4 (KPIs & Metrics)
- Extract current values from source assessments
- Retrieve previous period values for comparison
- Calculate status against targets
- Determine trends
- Document actions for underperforming KPIs

**Step 5: Consolidate Evidence (1 hour)**

- Open Sheet 5 (Evidence Register)
- Copy evidence entries from S1, S2, S3 evidence registers
- Verify each evidence item is accessible
- Update location/link fields if needed
- Identify any missing evidence

**Step 6: Quality Check (30 minutes)**

- Complete self-assessment using Quality Checklist below
- Verify all scores calculate correctly
- Verify gap counts match source assessments
- Verify evidence is accessible

**Step 7: Obtain Approvals (3-5 business days)**

- Present dashboard to Facilities Manager
- Address feedback, obtain approval
- Present to Physical Security Manager
- Present to IT Security Manager
- Present to CISO
- Present to Risk & Compliance Officer
- Document all approvals

**Step 8: Submit for Audit**

- Package dashboard with source assessments
- Prepare audit briefing materials
- Store in ISMS evidence repository

## Gap Prioritisation

### Severity Assignment Guidelines

**Critical Severity:**

- Server room accessible without MFA
- Perimeter breach points identified and not secured
- Access logs not retained (compliance violation)
- No visitor sign-in process
- Emergency exits not alarmed

**High Severity:**

- Clean desk compliance below 70%
- Access reviews overdue by more than 3 months
- Anti-tailgating measures not effective
- CCTV blind spots in sensitive areas
- Environmental monitoring alerts going to inactive mailbox

**Medium Severity:**

- Screen privacy filters not deployed
- Meeting room clearing procedures not followed
- Visitor pre-registration not utilised
- Shared facility perimeter unclear
- VC equipment not secured

### Remediation Planning

**Critical Gaps:**

- Assign immediately to senior owner
- Target remediation within 30 days
- Track weekly until closed
- Implement compensating controls if full remediation takes longer

**High Gaps:**

- Assign to appropriate owner
- Target remediation within 60 days
- Track bi-weekly until closed
- Document interim measures if needed

**Medium Gaps:**

- Assign to appropriate owner
- Target remediation within 90 days
- Track monthly until closed
- May be batched for efficiency

## Common Pitfalls

### Pitfall 1: Dashboard Before Assessments

- **Problem:** Completing dashboard before source assessments are finished

- **Impact:** Dashboard reflects incomplete or inaccurate data

- **How to Avoid:**
  - Verify S1, S2, S3 are all COMPLETE and APPROVED before starting dashboard
  - Check for Level 4 (CISO) approval on each source assessment
  - Do not accept "almost complete" as sufficient

- **Example:**
  - ❌ MISTAKE: Starting dashboard because S1 and S2 are complete but S3 is "nearly done"
  - ✅ CORRECT: Waiting for all three assessments to receive Level 4 approval before starting

### Pitfall 2: Incorrect Weighting

- **Problem:** Not properly weighting control compliance scores

- **Impact:** Overall score doesn't reflect relative importance of controls

- **How to Avoid:**
  - Use policy-defined weighting (if specified)
  - If not specified, use recommended weighting (35/40/25)
  - Document weighting methodology used

- **Example:**
  - ❌ MISTAKE: Simple average of three scores (33/33/33) when policy specifies different weights
  - ✅ CORRECT: Using policy-defined weights or documenting rationale for chosen weights

### Pitfall 3: Missing Gaps

- **Problem:** Not extracting all gaps from source assessments

- **Impact:** Incomplete remediation tracking, audit finding

- **How to Avoid:**
  - Systematically review each source assessment sheet
  - Extract EVERY Non-Compliant and Partial item
  - Cross-check gap count with source assessment summary

- **Example:**
  - ❌ MISTAKE: Only extracting gaps from Summary Dashboard, missing items in detailed sheets
  - ✅ CORRECT: Reviewing each row of each data entry sheet for Non-Compliant/Partial status

### Pitfall 4: Undefined Remediation Owners

- **Problem:** Gaps identified but no owner assigned or role assigned instead of name

- **Impact:** No accountability, gaps not remediated

- **How to Avoid:**
  - Assign specific named individual (not role) as owner for each gap
  - Verify owner has authority and capacity to remediate
  - Confirm owner acceptance

- **Example:**
  - ❌ MISTAKE: Owner field contains "IT Security" or "Facilities Team"
  - ✅ CORRECT: Owner field contains "John Smith - IT Security Manager"

### Pitfall 5: Unrealistic Target Dates

- **Problem:** Target dates that cannot be achieved

- **Impact:** Missed deadlines, credibility issues, audit findings

- **How to Avoid:**
  - Consult with remediation owners before setting dates
  - Consider resource availability and dependencies
  - Apply severity-based timeframes as guidance
  - Build in buffer for unexpected delays

- **Example:**
  - ❌ MISTAKE: Setting 30-day target for server room renovation requiring procurement
  - ✅ CORRECT: Consulting with owner, setting 90-day target with interim compensating control at 30 days

### Pitfall 6: Evidence Consolidation Incomplete

- **Problem:** Dashboard evidence register doesn't reference all source evidence

- **Impact:** Audit finding, incomplete documentation

- **How to Avoid:**
  - Systematically copy all entries from each source assessment
  - Verify total evidence count matches sum of sources
  - Check for duplicates and consolidate

- **Example:**
  - ❌ MISTAKE: Only including "key" evidence items
  - ✅ CORRECT: Including all 35 evidence items from across S1, S2, S3

### Pitfall 7: KPIs Not Updated

- **Problem:** KPIs showing outdated values from previous period

- **Impact:** Dashboard doesn't reflect current state

- **How to Avoid:**
  - Extract current values from source assessments
  - Don't reuse previous period values without verification
  - Document data freshness

- **Example:**
  - ❌ MISTAKE: Copying KPI values from previous dashboard without verification
  - ✅ CORRECT: Recalculating each KPI from current source assessments

### Pitfall 8: Approval Without Review

- **Problem:** Approvers signing without reviewing content

- **Impact:** Missed errors, no value from review process

- **How to Avoid:**
  - Present dashboard in meeting, allow questions
  - Provide summary briefing document
  - Document feedback and how addressed

- **Example:**
  - ❌ MISTAKE: Sending dashboard by email and receiving signature by reply
  - ✅ CORRECT: Scheduling 30-minute review meeting with each approver

### Pitfall 9: Gap Severity Misclassified

- **Problem:** Critical gaps marked as High or Medium

- **Impact:** Remediation delayed, risk underestimated

- **How to Avoid:**
  - Apply severity criteria consistently
  - Review high-impact items for Critical classification
  - Have second person review severity assignments

- **Example:**
  - ❌ MISTAKE: Server room with badge-only access marked as "Medium"
  - ✅ CORRECT: Server room with badge-only access marked as "Critical" (MFA required)

### Pitfall 10: Trends Not Calculated

- **Problem:** Trend column left blank or all marked "Stable"

- **Impact:** Unable to identify improving or declining areas

- **How to Avoid:**
  - Compare current values to previous period
  - Calculate percentage change
  - Apply threshold for trend classification

- **Example:**
  - ❌ MISTAKE: All KPIs marked "Stable" without comparison
  - ✅ CORRECT: Comparing to Q3 values, marking visitor compliance as "Declining" (down 10%)

### Pitfall 11: Source Assessment Versions Mixed

- **Problem:** Using different versions of source assessments

- **Impact:** Inconsistent data, calculation errors

- **How to Avoid:**
  - Use final approved version of each source assessment
  - Document version numbers and approval dates
  - Store copies of source assessments used

- **Example:**
  - ❌ MISTAKE: Using S1 draft while S2 and S3 are final approved
  - ✅ CORRECT: Using final approved versions of all three assessments

### Pitfall 12: Audit Trail Incomplete

- **Problem:** Cannot demonstrate how dashboard values were derived

- **Impact:** Audit finding, credibility issues

- **How to Avoid:**
  - Document source for each value
  - Include cell references or calculation methodology
  - Store source assessments with dashboard

- **Example:**
  - ❌ MISTAKE: Dashboard shows 85% compliance with no explanation of calculation
  - ✅ CORRECT: Notes section documents formula and source cell references

### Pitfall 13: Deferred Gaps Not Justified

- **Problem:** Gaps marked "Deferred" without documented justification

- **Impact:** Audit finding, perceived avoidance

- **How to Avoid:**
  - Document specific reason for deferral
  - Document compensating controls in place
  - Set review date for deferred gaps
  - Require management approval for deferrals

- **Example:**
  - ❌ MISTAKE: Gap marked "Deferred" with no explanation
  - ✅ CORRECT: Gap marked "Deferred" with note: "Pending building renovation Q3 2026, compensating control: additional security patrols"

### Pitfall 14: Previous Period Not Comparable

- **Problem:** Comparing to previous period with different scope

- **Impact:** Invalid trend analysis

- **How to Avoid:**
  - Ensure scope consistency between periods
  - Document any scope changes
  - Normalise values if scope changed

- **Example:**
  - ❌ MISTAKE: Comparing Q1 (3 buildings) to Q4 (5 buildings) without adjustment
  - ✅ CORRECT: Noting scope change, comparing only consistent subset or adjusting metrics

### Pitfall 15: Evidence Accessibility Not Verified

- **Problem:** Evidence links broken or inaccessible

- **Impact:** Audit finding, cannot verify compliance

- **How to Avoid:**
  - Test each evidence link
  - Verify permissions allow auditor access
  - Update broken links immediately

- **Example:**
  - ❌ MISTAKE: Evidence register lists SharePoint path but file was moved
  - ✅ CORRECT: Testing each link, updating path for moved files

### Pitfall 16: Approver Sequence Not Followed

- **Problem:** Approvals obtained out of sequence

- **Impact:** Later approvers miss earlier corrections

- **How to Avoid:**
  - Follow defined approval sequence
  - Document any deviations from sequence
  - Re-present to earlier approvers if significant changes made

- **Example:**
  - ❌ MISTAKE: Getting CISO approval before Facilities Manager review
  - ✅ CORRECT: Following sequence: Facilities, Physical Security, IT Security, CISO, Risk & Compliance

## Review & Approval

### Five-Level Approval Workflow

**Level 1: Facilities Manager**

- Review physical infrastructure accuracy
- Verify perimeter and building controls
- Confirm shared facility arrangements
- Date and sign

**Level 2: Physical Security Manager**

- Review security control accuracy
- Verify entry controls and secure areas
- Confirm gap prioritisation
- Date and sign

**Level 3: IT Security Manager**

- Review integration and technology accuracy
- Verify server rooms and access systems
- Confirm technical gap remediation plans
- Date and sign

**Level 4: CISO**

- Executive review and approval
- Resource allocation for gap remediation
- Strategic direction confirmation
- Date and sign

**Level 5: Risk & Compliance Officer**

- Audit readiness certification
- Gap remediation plan adequacy
- Evidence completeness
- Date and sign

**Timeline:** 3-5 business days for all five levels

### Escalation Procedures

If approval is delayed beyond 5 business days:

1. Send reminder to current approver
2. Escalate to approver's manager after 3 additional days
3. Escalate to CISO if critical gaps remain unaddressed beyond 10 days
