## Assessment Process Overview

### Purpose & Scope

**Assessment Name:** ISMS-IMP-A.7.6-7-14-S4 - Compliance Dashboard & Assessment Process

#### What This Assessment Framework Covers

This is the **CONSOLIDATED assessment framework** that brings together Controls A.7.6, A.7.7, and A.7.14 into a unified compliance view. This framework defines:

- **Assessment frequency:** Monthly metrics, quarterly full assessment
- **Assessment process:** How to conduct each assessment type
- **Dashboard integration:** How individual assessments (S1, S2, S3) feed into unified dashboard
- **Reporting:** Monthly summary, quarterly comprehensive report
- **Evidence framework:** Audit-ready evidence collection across all secure areas and media handling

#### Key Principle

This is NOT a separate assessment workbook - it's the **PROCESS** for conducting assessments and the **DASHBOARD** that consolidates results from:

- ISMS-IMP-A.7.6-7-14-S1 (Secure Areas Working Assessment)
- ISMS-IMP-A.7.6-7-14-S2 (Clear Desk/Screen Compliance)
- ISMS-IMP-A.7.6-7-14-S3 (Equipment Disposal Assessment)

#### How This Integrates with Individual Assessments

| Assessment | Focus | Frequency | Feeds Into Dashboard |
|------------|-------|-----------|---------------------|
| ISMS-IMP-A.7.6-7-14-S1 | Secure Areas (A.7.6) | Quarterly | Secure area compliance score, third-party access metrics |
| ISMS-IMP-A.7.6-7-14-S2 | Clear Desk/Screen (A.7.7) | Monthly audits, Quarterly full | Clear desk audit pass rate, screen lock compliance |
| ISMS-IMP-A.7.6-7-14-S3 | Equipment Disposal (A.7.14) | Semi-annual + per event | Disposal compliance, certificate collection rate |
| **ISMS-IMP-A.7.6-7-14-S4** | **Compliance Dashboard** | **Monthly + Quarterly** | **Overall compliance score, trends, gap tracking** |

### Assessment Types

#### Monthly Metrics Update

**Duration:** 1-2 hours
**Frequency:** First week of each month
**Scope:** Key metrics only - no full assessment required

**What's Updated:**

- Clear desk audit results (from monthly audits)
- Screen lock compliance (from endpoint management)
- Secure area incidents (from incident management)
- Disposal events (from disposal log)

**Deliverable:** Dashboard update + 1-page monthly summary

#### Quarterly Full Assessment

**Duration:** 4-8 hours (coordination of S1, S2, S3 assessments)
**Frequency:** End of quarter
**Scope:** Full assessment of all three controls

**What's Assessed:**

- Complete ISMS-IMP-A.7.6-7-14-S1 (Secure Areas Working) assessment
- Complete ISMS-IMP-A.7.6-7-14-S2 (Clear Desk/Screen Compliance) assessment
- Update ISMS-IMP-A.7.6-7-14-S3 (Equipment Disposal) for quarter
- Consolidate scores into dashboard
- Generate quarterly report

**Deliverable:** Updated S1, S2, S3 workbooks + Quarterly compliance report + Updated dashboard

### Who Conducts Assessments

#### Assessment Roles

**Monthly:**

- Facilities Manager (clear desk audit results)
- IT Operations (screen lock compliance, disposal events)

**Quarterly:**

- Facilities Manager + Security Operations (S1 - Secure Areas)
- Facilities Manager + IT Operations (S2 - Clear Desk/Screen)
- IT Operations + Asset Management (S3 - Equipment Disposal)

**Approver:**

- CISO (all assessments)

### Expected Outputs

Upon completion of this assessment framework, you will have:

1. **Monthly dashboard updates** - Tracking key metrics month-over-month
2. **Quarterly compliance reports** - Comprehensive assessment results
3. **Consolidated compliance score** - Single score for Controls A.7.6/A.7.7/A.7.14
4. **Trend analysis** - 12-month trends identifying patterns
5. **Gap remediation tracking** - Status of all non-compliant items
6. **Evidence register** - All supporting documentation organised for audit
7. **Executive reporting** - High-level summaries for governance

## Completion Walkthrough

### Getting Started

#### Step 1: Understand the Dashboard Framework

Before starting any dashboard update, ensure you understand:

1. **The three source assessments:**
   - ISMS-IMP-A.7.6-7-14-S1 (Secure Areas Working) - Quarterly
   - ISMS-IMP-A.7.6-7-14-S2 (Clear Desk/Screen Compliance) - Monthly/Quarterly
   - ISMS-IMP-A.7.6-7-14-S3 (Equipment Disposal) - Semi-annual/Per Event

2. **The dashboard purpose:**
   - Consolidates individual assessment scores into unified compliance view
   - Tracks trends over time (month-over-month, quarter-over-quarter)
   - Provides executive-level reporting
   - Maintains audit-ready evidence framework

3. **Your update frequency:**
   - Monthly: Key metrics only (1-2 hours)
   - Quarterly: Full consolidation with S1, S2, S3 integration (4-8 hours)

#### Step 2: Verify Prerequisites

Before opening the dashboard workbook, confirm:

- [ ] You have access to the ISMS Evidence Library (SharePoint/Confluence)
- [ ] You have read permissions for S1, S2, S3 assessment workbooks
- [ ] You have received the latest reports from:
  - Facilities Manager (clear desk audit results)
  - IT Operations (screen lock compliance, disposal events)
  - Security Operations (secure area incidents)
- [ ] Previous month/quarter dashboard is archived

#### Step 3: Open and Prepare Workbook

1. Navigate to: `\\fileserver\isms\assessments\A.7.6-7-14\templates\`
2. Copy: `ISMS_Dashboard_Secure_Areas_Media_TEMPLATE.xlsx`
3. Rename to: `ISMS-IMP-A.7.6-7-14.S4_Compliance_Dashboard_YYYYMMDD.xlsx`
4. Open workbook in Microsoft Excel
5. Enable editing if prompted
6. Navigate to Sheet 1 (Instructions & Legend)
7. Complete metadata fields:
   - Generated By: [Your Name]
   - Organisation: [Organisation Name]
   - Reporting Period: [Month YYYY] or [QYYYY]

#### Step 4: Navigate the Workbook Structure

**Monthly Update Flow:**
```
Sheet 1 (Metadata) → Sheet 2 (Executive Dashboard) → Sheet 4 (KPIs & Metrics)
```

**Quarterly Update Flow:**
```
Sheet 1 (Metadata) → Sheet 2 (Executive Dashboard) → Sheet 3 (Gap Analysis)
→ Sheet 4 (KPIs & Metrics) → Sheet 5 (Evidence Register) → Sheet 6 (Approval)
```

### Completing Sheet 2: Executive Dashboard

**For Monthly Updates:**

1. Locate the yellow input cells in the Key Metrics section
2. Enter each metric:
   - Clear Desk Audit Pass Rate: Enter percentage from audit report
   - Screen Lock Compliance: Enter percentage from endpoint management
   - Secure Area Incidents: Enter count from incident log
   - Disposal Certificate Rate: Enter percentage (certificates / required)

3. Verify formulas calculate Overall Compliance Score automatically
4. Check conditional formatting shows correct status colours

**For Quarterly Updates:**

1. Import compliance scores from completed assessments:
   - A.7.6 Score: Copy from S1 Summary Dashboard "Overall Compliance" cell
   - A.7.7 Score: Copy from S2 Summary Dashboard "Overall Compliance" cell
   - A.7.14 Score: Copy from S3 Summary Dashboard "Overall Compliance" cell

2. Update Critical Gaps count from each assessment
3. Update High Priority count from each assessment
4. Verify Overall Compliance Score calculates correctly using weighting formula

### Completing Sheet 3: Gap Analysis (Quarterly Only)

1. Export gap findings from S1, S2, S3 assessments
2. Assign unique Gap IDs (GAP-YYYY-QX-NNN format)
3. For each gap:
   - Control: Identify which control (A.7.6, A.7.7, A.7.14)
   - Finding: Clear description of non-compliance
   - Severity: Critical, High, Medium, or Low
   - Owner: Assign responsible person/role
   - Target Date: Set based on severity (Critical=30d, High=60d, Medium=90d)
   - Status: Open (for new gaps)

4. Review gaps from previous quarter:
   - Update Status to "Closed" if remediated (with evidence)
   - Update Status to "In Progress" if partially complete
   - Escalate to CISO if overdue

### Completing Sheet 4: KPIs & Metrics

1. Update Current value for each KPI
2. Verify Target values are correct (do not reduce targets)
3. Status will auto-calculate (Pass/Fail)
4. Add Trend indicator (↑ Improving, → Stable, ↓ Declining)
5. Investigate any declining trends and document in Notes

### Completing Sheet 5: Evidence Register (Quarterly Only)

1. Consolidate evidence from S1, S2, S3 assessments
2. Assign unique Evidence IDs (EVD-YYYY-QX-NNN format)
3. For each evidence item:
   - Evidence Type: Document, Screenshot, Log, Certificate, etc.
   - Description: Clear description of what evidence demonstrates
   - Related Sheet: Reference to source assessment
   - File Name: Exact filename
   - File Location: Full path in ISMS Evidence Library
   - Collection Date: Date evidence was gathered
   - Collected By: Person who collected evidence

4. Verify all files exist in documented locations

### Completing Sheet 6: Approval & Sign-Off (Quarterly Only)

1. Complete Level 1 (Assessor) section:
   - Enter your name
   - Add signature (or electronic approval reference)
   - Enter date
   - Add any comments on assessment quality

2. Submit to Level 2 (CISO) for review:
   - Email dashboard with quarterly report attached
   - Request approval within 5 business days

3. After Level 2 approval, submit to Level 3 (Compliance Officer):
   - Include CISO approval confirmation
   - Request audit readiness certification

## Monthly Metrics Update

### Purpose

Collect **key metrics** without requiring full assessment. This provides continuous compliance monitoring between quarterly full assessments.

### Timeline

**Frequency:** First week of each month (assess previous month's data)
**Duration:** 1-2 hours

### Data Collection Process

#### Step 1: Clear Desk Audit Results (Control A.7.7)

**What to Collect:**

- Number of clear desk audits conducted in previous month
- Number of workstations audited
- Number compliant / non-compliant
- Common issues found

**Calculate:**

- Audit pass rate: (Compliant / Total Audited) × 100%
- Target: >95%

**Document in Dashboard:**

- Clear Desk Audit Pass Rate: [XX%]

#### Step 2: Screen Lock Compliance (Control A.7.7)

**What to Collect:**

- Endpoint management compliance report
- Number of devices with compliant screen lock settings
- Number of devices non-compliant

**Calculate:**

- Screen lock compliance: (Compliant Devices / Total Devices) × 100%
- Target: 100%

**Document in Dashboard:**

- Screen Lock Compliance: [XX%]

#### Step 3: Secure Area Incidents (Control A.7.6)

**What to Collect:**

- Query incident management for secure area incidents (previous month)
- Count by type (unauthorised access, procedure violation, etc.)

**Document in Dashboard:**

- Secure Area Incidents: [Count] (Target: 0)

#### Step 4: Equipment Disposals (Control A.7.14)

**What to Collect:**

- Number of equipment disposals in previous month
- Number with certificates obtained (where required)

**Calculate:**

- Certificate collection rate: (Certificates / Required) × 100%
- Target: 100% for CONFIDENTIAL equipment

**Document in Dashboard:**

- Equipment Disposed: [Count]
- Certificates Collected: [XX%]

### Dashboard Update Process

1. Open Dashboard Workbook: ISMS_Dashboard_Secure_Areas_Media_YYYY-MM.xlsx
2. Navigate to "Executive Dashboard" sheet
3. Update yellow input cells with monthly metrics
4. Dashboard auto-calculates overall compliance score
5. Save as new month's dashboard
6. Generate monthly summary report

### Monthly Reporting

**Report Format:** Email body or 1-page summary

**Template:**


**Distribution:**

- Email to: CISO, Facilities Manager, IT Operations Manager

**Timeline:** Complete by 7th day of month

## Quarterly Full Assessment

### Purpose

Conduct **comprehensive assessment** of all three controls with full workbook updates for S1, S2, and S3.

### Timeline

**Frequency:** End of quarter (March 31, June 30, September 30, December 31)
**Duration:** 4-8 hours

### Assessment Process

#### Week Before Quarter-End: Preparation

1. Schedule assessment activities
2. Prepare S1, S2, S3 workbooks (copy from templates or previous quarter)
3. Gather data for each assessment
4. Review previous quarter's findings (verify remediation actions completed)

#### Quarter-End Week: Assessment Execution

**Day 1-2: Complete Individual Assessments**

Follow the detailed procedures in:

- **ISMS-IMP-A.7.6-7-14-S1** (Secure Areas Working) - Complete all sheets
- **ISMS-IMP-A.7.6-7-14-S2** (Clear Desk/Screen Compliance) - Complete all sheets
- **ISMS-IMP-A.7.6-7-14-S3** (Equipment Disposal) - Update for quarter

**Output:** Three updated assessment workbooks with compliance scores

**Day 3: Dashboard Consolidation**

1. Open quarterly dashboard workbook
2. Import compliance scores from S1, S2, S3:
   - Secure Areas Score (from S1 Summary Dashboard)
   - Clear Desk/Screen Score (from S2 Summary Dashboard)
   - Equipment Disposal Score (from S3 Summary Dashboard)
3. Dashboard auto-calculates overall compliance score
4. Review trend charts (current quarter vs. previous quarters)
5. Document gaps identified

#### Week After Quarter-End: Reporting

**Generate Quarterly Report (5-10 pages):**

1. **Executive Summary** (1 page)
   - Overall compliance score: [XX%]
   - Quarter-over-quarter trend
   - Key findings and recommendations

2. **Assessment Results by Control** (2-4 pages)
   - **Control A.7.6 (Secure Areas):** [XX%]
   - **Control A.7.7 (Clear Desk/Screen):** [XX%]
   - **Control A.7.14 (Equipment Disposal):** [XX%]

3. **Detailed Findings** (1-2 pages)
   - Non-compliant items
   - Root causes

4. **Remediation Plan** (1-2 pages)
   - Immediate actions
   - Short-term actions
   - Resource requirements

5. **Appendices**
   - Compliance scores detail
   - Evidence register summary

**Distribution:**

- CISO (primary)
- Executive Management
- Facilities Manager
- IT Operations Manager
- Internal Audit

**Timeline:** Complete within 10 days after quarter-end

## Common Pitfalls

### Pitfall 1: Incomplete Data Consolidation

❌ **MISTAKE:** Only importing partial data from S1, S2, or S3 assessments into the dashboard

**Impact:** Skewed compliance scores that do not reflect actual organisational status

**How to Avoid:**
- Verify ALL sheets from each assessment are reviewed before importing scores
- Cross-check individual assessment totals against dashboard entries
- Document any exclusions with clear justification

### Pitfall 2: Ignoring Trend Analysis

❌ **MISTAKE:** Focusing only on current quarter scores without comparing to historical data

**Impact:** Missing patterns that indicate systemic issues or improvement opportunities

**How to Avoid:**
- Always populate trend columns comparing current to previous quarters
- Investigate any significant changes (>5% variance) between periods
- Document root causes for both improvements and declines

### Pitfall 3: Overdue Gap Remediation

❌ **MISTAKE:** Carrying forward unresolved gaps quarter after quarter without escalation

**Impact:** Chronic non-compliance, audit findings, potential security incidents

**How to Avoid:**
- Set hard deadlines for gap remediation (Critical: 30 days, High: 60 days)
- Escalate overdue items to CISO automatically after target date
- Document remediation blockers and resource requirements

### Pitfall 4: Inconsistent Scoring Methodology

❌ **MISTAKE:** Different assessors using different criteria when scoring compliance

**Impact:** Unreliable metrics, inability to compare across periods or business units

**How to Avoid:**
- Use this specification as the single source of truth for scoring
- Train all assessors on consistent interpretation of Compliant/Partial/Non-Compliant
- Have CISO validate scoring methodology annually

### Pitfall 5: Missing Evidence Links

❌ **MISTAKE:** Recording compliance scores without corresponding evidence references

**Impact:** Cannot substantiate claims during audit, assessment credibility undermined

**How to Avoid:**
- Every metric entry must have corresponding evidence in the Evidence Register
- Use consistent evidence ID referencing (e.g., EVD-2026-Q1-001)
- Verify evidence actually exists in documented location before finalising

### Pitfall 6: Dashboard Calculation Errors

❌ **MISTAKE:** Manually overriding formula cells or entering values in calculated fields

**Impact:** Broken calculations, incorrect compliance scores, audit discrepancies

**How to Avoid:**
- Never modify grey/formula cells - only update yellow input cells
- If formulas appear broken, regenerate workbook from template
- Validate dashboard totals against source assessment scores

### Pitfall 7: Incomplete Monthly Updates

❌ **MISTAKE:** Skipping monthly updates when "nothing significant happened"

**Impact:** Gaps in continuous monitoring evidence, inability to demonstrate ongoing compliance

**How to Avoid:**
- Perform monthly update even if all metrics show 100% compliance
- Document "No incidents" and "No changes" explicitly
- Maintain audit trail of all monthly review dates

### Pitfall 8: Misaligned Reporting Periods

❌ **MISTAKE:** Using different date ranges for S1, S2, S3 assessments within same quarterly cycle

**Impact:** Dashboard consolidation reflects inconsistent timeframes, metrics not comparable

**How to Avoid:**
- Define fixed quarterly assessment periods (e.g., Q1 = Jan 1 - Mar 31)
- All three assessments must use identical date range
- Document assessment period clearly in each workbook header

### Pitfall 9: Stakeholder Distribution Gaps

❌ **MISTAKE:** Sending dashboard reports only to CISO, excluding operational stakeholders

**Impact:** Facilities and IT Operations unaware of gaps affecting their areas

**How to Avoid:**
- Follow distribution list in this specification for both monthly and quarterly reports
- Confirm receipt from all required stakeholders
- Maintain distribution log as evidence

### Pitfall 10: Weighting Rationale Not Documented

❌ **MISTAKE:** Using default weighting without considering organisational risk profile

**Impact:** Overall compliance score may not reflect actual risk exposure

**How to Avoid:**
- Review weighting formula annually with CISO
- Document rationale for any weighting adjustments
- Ensure weighting reflects current business priorities and threat landscape

### Pitfall 11: Delayed Incident Integration

❌ **MISTAKE:** Waiting until quarterly assessment to record secure area incidents

**Impact:** Incidents not reflected in monthly metrics, delayed remediation

**How to Avoid:**
- Integrate incident data into dashboard within 5 business days of incident closure
- Include incident trends in monthly summary even between quarterly assessments
- Flag any Critical/High incidents immediately to CISO

### Pitfall 12: KPI Target Drift

❌ **MISTAKE:** Accepting reduced targets when compliance consistently falls short

**Impact:** Lowered standards, normalisation of non-compliance

**How to Avoid:**
- Targets should be maintained or increased, never reduced without CISO approval
- Document justification if any target is temporarily relaxed
- Include remediation plan to return to original targets

### Pitfall 13: Executive Summary Overload

❌ **MISTAKE:** Including excessive technical detail in executive presentations

**Impact:** Executives disengage, key messages lost, decisions delayed

**How to Avoid:**
- Executive Dashboard should fit on one screen/page
- Limit to: Overall score, trend, top 3 findings, required decisions
- Keep detailed analysis in supporting sheets

### Pitfall 14: Orphaned Evidence Files

❌ **MISTAKE:** Evidence files stored in personal drives or email attachments

**Impact:** Evidence unavailable for audit, no version control, potential data loss

**How to Avoid:**
- All evidence must be stored in designated ISMS SharePoint/Confluence location
- Use folder structure specified in Audit Evidence Framework
- Verify file accessibility from shared location

### Pitfall 15: Approval Workflow Shortcuts

❌ **MISTAKE:** Self-approving at multiple levels or obtaining approvals out of sequence

**Impact:** Audit finding for inadequate segregation of duties

**How to Avoid:**
- Each approval level must be different individual
- Approvals must follow defined sequence (Assessor → CISO → Compliance Officer)
- Document approval dates and ensure sequential order

### Pitfall 16: Quarterly Report Scope Creep

❌ **MISTAKE:** Including unrelated topics or excessive recommendations in quarterly reports

**Impact:** Core compliance findings diluted, action items unclear

**How to Avoid:**
- Follow report template structure exactly (5-10 pages maximum)
- Limit recommendations to those directly addressing identified gaps
- Separate strategic recommendations for annual review process
