## 1. Purpose and Scope

### 1.1 Assessment Framework Purpose

This document defines the **assessment methodology and evidence framework** for verifying compliance with the Capacity Management policy (ISMS-POL-A.8.6) and demonstrating effective implementation of ISO/IEC 27001:2022 Control A.8.6.

**Assessment Objectives**:

- **Measure Compliance**: Verify capacity management requirements are implemented as defined in ISMS-POL-A.8.6-S2
- **Assess Effectiveness**: Evaluate whether capacity management activities achieve their intended objectives (prevent outages, enable planning, optimize costs)
- **Identify Gaps**: Discover areas where capacity management implementation is incomplete or ineffective
- **Support Improvement**: Provide data-driven insights for continuous improvement of capacity management practices
- **Enable Audit**: Generate objective evidence for internal audits, external audits, and ISO 27001 certification

### 1.2 Assessment Scope

This framework defines assessment methodologies for:

1. **Assessment 1: Capacity Utilization Analysis** - Current state capacity inventory and utilization status
2. **Assessment 2: Capacity Forecasts and Planning** - Future state projections and capacity planning effectiveness
3. **Dashboard: Capacity Management Overview** - Consolidated executive view of capacity health and maturity

**Assessment Frequency**:
- **Assessment 1 (Utilization)**: Monthly (operational monitoring)
- **Assessment 2 (Forecasts)**: Quarterly (capacity planning cycles)
- **Dashboard**: Monthly (executive reporting)
- **Comprehensive Assessment**: Annually (ISO 27001 audit preparation)

### 1.3 Assessment Approach

The assessment framework uses **automated assessment tools** (Excel workbooks generated by Python scripts) to ensure:

- **Consistency**: Standardized assessment methodology across all assessments
- **Repeatability**: Same assessment can be executed consistently over time
- **Maintainability**: Python scripts can be updated as requirements evolve
- **Auditability**: Assessment results are documented and traceable
- **Efficiency**: Automated workbook generation reduces manual effort

**Systems Engineering Methodology**: Assessments are data-driven, metric-based, and objectively verifiable rather than subjective opinion-based evaluations.

## 2. Assessment 1: Capacity Utilization Analysis

### 2.1 Assessment Purpose

**Objective**: Document current capacity inventory and utilization status across all monitored resources to verify:
- Monitoring coverage completeness
- Current utilization levels relative to thresholds
- Resources requiring immediate attention
- Capacity headroom available for growth

**Assessment Output**: Excel workbook (generated by `generate_assessment_1_capacity_utilization.py`)

### 2.2 Assessment Methodology

#### 2.2.1 Resource Inventory

**Process**:
1. Enumerate all resources within scope per ISMS-POL-A.8.6-S2, Section 2:
   - Compute resources (physical servers, VMs, containers, cloud instances)
   - Storage resources (disks, volumes, databases, backup, archive)
   - Network resources (links, switches, load balancers, firewalls)
   - Application resources (user capacity, transaction capacity)
   - Cloud service capacity (quotas, limits)

2. For each resource, document:
   - Resource identification (name, type, location, business function)
   - Total capacity (maximum capacity available)
   - Current utilization (actual usage at assessment time)
   - Peak utilization (maximum observed in last 30 days)
   - Utilization percentage (current and peak)
   - Threshold status (OK, Warning, Critical)

3. Document monitoring status:
   - Is monitoring configured? (Yes/No)
   - Monitoring tool used
   - Monitoring data available? (Yes/No)

**Data Sources**:
- Monitoring tool exports (Prometheus, Datadog, CloudWatch, etc.)
- Infrastructure management systems (VMware vCenter, AWS Console, Azure Portal)
- Manual inventory (for resources not yet monitored)
- CMDB (Configuration Management Database) if available

#### 2.2.2 Utilization Analysis

**Process**:
1. Calculate utilization percentage: (Used Capacity / Total Capacity) × 100%
2. Classify each resource by threshold status:
   - **OK**: Utilization < Warning Threshold (typically <70%)
   - **Warning**: Warning Threshold ≤ Utilization < Critical Threshold (typically 70-85%)
   - **Critical**: Utilization ≥ Critical Threshold (typically ≥85%)
3. Calculate capacity headroom: Total Capacity - Current Utilization
4. Identify resources requiring action:
   - Resources at Critical threshold → Immediate capacity expansion required
   - Resources at Warning threshold → Begin capacity planning
   - Resources consistently at low utilization (<30%) → Right-sizing candidates

**Threshold Application**:
Apply thresholds per ISMS-POL-A.8.6-S2, Section 4.2:
- CPU: Warning 70%, Critical 85%
- Memory: Warning 75%, Critical 90%
- Disk Space: Warning 75%, Critical 85%
- Network Bandwidth: Warning 70%, Critical 85%
- Application Sessions: Warning 75%, Critical 90%

**Custom Thresholds**: Allow for resource-specific threshold overrides based on business criticality or workload characteristics.

#### 2.2.3 Coverage Assessment

**Process**:
1. Calculate monitoring coverage percentage:
   - Coverage % = (Monitored Resources / Total Resources) × 100%
2. Identify monitoring gaps:
   - Resources in scope but not monitored
   - Resources monitored but with incomplete metrics
3. Document reasons for non-coverage:
   - Planned (in implementation pipeline)
   - Technical constraints (legacy systems, monitoring limitations)
   - Exceptions (approved policy exceptions)

**Coverage Target**: 100% of production systems, 90% of non-production systems per ISMS-POL-A.8.6-S2, Section 3.1.

### 2.3 Assessment Workbook Structure

**Excel Workbook Sheets**:

1. **Instructions & Legend** - Assessment instructions and status indicators
2. **Compute_Resources** - CPU, memory, VM/container capacity
3. **Storage_Resources** - Disk, database, backup capacity
4. **Network_Resources** - Bandwidth, throughput, network services capacity
5. **Application_Resources** - User sessions, transactions, queues
6. **Cloud_Resources** - Cloud quotas, limits, instance counts
7. **Threshold_Summary** - Summary of resources by threshold status
8. **Coverage_Analysis** - Monitoring coverage assessment
9. **Evidence_Register** - Supporting documentation index

**Key Features**:

- Conditional formatting (color-coding by threshold status)
- Dropdown validations for consistency
- Summary statistics and charts
- UTF-8 emoji support (✅ ⚠️ ❌ 📊)

**Required Evidence** for Assessment 1:

| Evidence Type | Examples | Purpose |
|---------------|----------|---------|

## 3. Assessment 2: Capacity Forecasts and Planning

### 3.1 Assessment Purpose

**Objective**: Document capacity forecasts, validate forecast accuracy, and track capacity planning effectiveness to verify:
- Forecasts are developed using data-driven methodologies
- Forecast accuracy meets acceptable targets
- Capacity planning is proactive rather than reactive
- Capacity expansions are planned with adequate lead time

**Assessment Output**: Excel workbook (generated by `generate_assessment_2_capacity_forecasts.py`)

### 3.2 Assessment Methodology

#### 3.2.1 Historical Utilization Analysis

**Process**:
1. Collect historical utilization data (minimum 6 months, preferably 12-24 months)
2. Document for each resource:
   - Historical utilization values (monthly or quarterly data points)
   - Utilization trend (increasing, stable, decreasing)
   - Seasonal patterns (if identified)
   - Anomalies or one-time events (to be excluded from trend analysis)

**Data Sources**:
- Monitoring tool historical data exports
- Previous capacity assessment reports
- Infrastructure management system reports

#### 3.2.2 Trend Analysis and Forecasting

**Process**:
1. Perform statistical trend analysis:
   - **Linear Regression**: Fit trend line to historical data
   - **Growth Rate Calculation**: Calculate % growth per month/quarter/year
   - **Seasonal Adjustment**: Account for recurring patterns if present
   
2. Calculate forecasted utilization:
   - **6-month forecast**: Short-term tactical planning
   - **12-month forecast**: Annual budget planning
   - **24-month forecast** (optional): Strategic planning

3. Document forecast methodology:
   - Method used (linear regression, exponential growth, manual projection)
   - Assumptions (business growth rate, seasonal factors, planned initiatives)
   - Confidence level (high/medium/low based on historical data quality)

**Forecasting Formula** (Linear Regression):
```
Forecasted Utilization = Current Utilization + (Growth Rate × Time Period)
```

**Growth Rate Calculation**:
```
Monthly Growth Rate = [(Latest Month - 12 Months Ago) / 12 Months Ago] / 12
```

#### 3.2.3 Capacity Exhaustion Projection

**Process**:
1. For each resource, calculate projected capacity exhaustion date:
   - If forecast exceeds Critical Threshold, calculate when this occurs
   - If forecast exceeds Maximum Capacity (100%), calculate when this occurs

2. Classify urgency:
   - **Immediate** (0-3 months): Urgent capacity expansion required
   - **Short-term** (3-6 months): Begin capacity planning now
   - **Medium-term** (6-12 months): Include in quarterly/annual planning
   - **Long-term** (>12 months): Monitor in strategic planning

**Exhaustion Date Calculation**:
```
Months Until Exhaustion = (Threshold - Current Utilization) / Monthly Growth Rate
Exhaustion Date = Current Date + Months Until Exhaustion
```

#### 3.2.4 Capacity Expansion Planning

**Process**:
1. Document planned capacity expansions:
   - Resource to expand
   - Capacity increment (how much to add)
   - Planned date (when expansion will occur)
   - Status (planned, in progress, completed)
   - Lead time (time from decision to deployment)
   - Cost (CapEx or OpEx)

2. Track capacity expansion execution:
   - Compare planned vs. actual dates
   - Document delays and reasons
   - Measure effectiveness (did expansion provide adequate headroom?)

3. Classify expansions:
   - **Proactive**: Planned in advance based on forecasts
   - **Reactive**: Emergency expansion due to threshold breach or exhaustion

**Proactive Expansion Ratio** = (Proactive Expansions / Total Expansions) × 100%
**Target**: ≥90% per ISMS-POL-A.8.6-S2, Section 1.2

#### 3.2.5 Forecast Accuracy Validation

**Process**:
1. Compare previous forecasts to actual observed utilization:
   - Retrieve forecasts made 3, 6, 12 months ago
   - Compare to actual utilization today
   - Calculate forecast error

2. Calculate accuracy metrics:
   - **Absolute Error**: |Forecasted - Actual|
   - **Percentage Error**: (|Forecasted - Actual| / Actual) × 100%
   - **Mean Absolute Percentage Error (MAPE)**: Average of percentage errors across all forecasts

3. Classify forecast accuracy:
   - **Excellent**: ≤5% error
   - **Good**: ≤10% error
   - **Acceptable**: ≤15% error
   - **Poor**: >15% error (requires methodology review)

**Accuracy Target**: ≤15% per ISMS-POL-A.8.6-S2, Section 5.3

4. Identify forecast bias:
   - **Over-forecasting**: Consistently predicting higher than actual (leads to over-provisioning)
   - **Under-forecasting**: Consistently predicting lower than actual (leads to capacity shortages)
   - **Unbiased**: Forecasts evenly distributed around actual

5. Continuous improvement:
   - Adjust forecasting methodology if accuracy is poor
   - Update growth rate assumptions based on actual trends
   - Incorporate feedback from business on planned initiatives

### 3.3 Assessment Workbook Structure

**Excel Workbook Sheets**:

1. **Instructions & Legend** - Forecasting methodology and instructions
2. **Historical_Utilization** - Historical data for trend analysis (12-24 months)
3. **Trend_Analysis** - Growth rates, patterns, trend lines
4. **Capacity_Forecasts** - 6-month and 12-month forecasts per resource
5. **Capacity_Exhaustion** - Projected exhaustion dates and urgency classification
6. **Planned_Expansions** - Capacity expansion schedule and tracking
7. **Forecast_Accuracy** - Validation of previous forecasts vs. actual
8. **Budget_Planning** - CapEx/OpEx impact of capacity expansions
9. **Evidence_Register** - Supporting documentation index
10. **Approval_Sign_Off** - Forecast approval and sign-off

**Key Features**:
- Trend line calculations (linear regression)
- Growth rate auto-calculation
- Exhaustion date projection
- Forecast vs. actual comparison charts
- Budget impact summaries

### 3.4 Evidence Requirements

**Required Evidence** for Assessment 2:

| Evidence Type | Examples | Purpose |
|---------------|----------|---------|
| Historical Monitoring Data | 12-24 months of utilization metrics | Support trend analysis |
| Forecasting Calculations | Spreadsheets, statistical analysis, trend lines | Verify forecasting methodology |

| Previous Forecast Reports | Quarterly capacity forecasts from prior periods | Enable forecast accuracy validation |

**Evidence Location**: Document in Evidence_Register sheet with references to stored evidence

**Frequency**: Quarterly (aligned with capacity planning cycle)

## 4. Dashboard: Capacity Management Overview

### 4.1 Dashboard Purpose

**Objective**: Provide executive-level consolidated view of capacity management health and maturity, integrating data from Assessments 1 and 2 to enable:
- Quick assessment of overall capacity status
- Identification of high-risk areas requiring attention
- Tracking of capacity management program effectiveness
- Communication with executive leadership and board

**Dashboard Output**: Excel workbook (generated by `generate_dashboard_capacity_management.py`)

### 4.2 Dashboard Methodology

#### 4.2.1 Capacity Health Score

**Process**:
1. Aggregate data from Assessment 1 (Utilization):
   - Total resources monitored
   - Resources at OK status (below warning threshold)
   - Resources at Warning status
   - Resources at Critical status

2. Calculate Capacity Health Score:
   ```
   Health Score = (Resources at OK Status / Total Resources) × 100%
   ```

3. Classify overall health:
   - **Excellent** (≥95%): Minimal capacity concerns
   - **Good** (85-94%): Some resources approaching thresholds
   - **Fair** (70-84%): Significant resources at warning/critical
   - **Poor** (<70%): Widespread capacity issues

**Health Score Target**: ≥95% per ISMS-POL-A.8.6-S2, Section 1.2

#### 4.2.2 Forecast Risk Assessment

**Process**:
1. Aggregate forecast data from Assessment 2:
   - Resources with exhaustion projected within 3 months (high risk)
   - Resources with exhaustion projected within 3-6 months (medium risk)
   - Resources with exhaustion projected within 6-12 months (low risk)
   - Resources with no exhaustion projected within 12 months (no risk)

2. Calculate risk distribution:
   - % High Risk resources
   - % Medium Risk resources
   - % Low Risk resources

3. Identify top capacity risks (top 10 resources by urgency)

#### 4.2.3 Capacity Planning Effectiveness

**Process**:
1. Calculate proactive vs. reactive expansion ratio:
   - Proactive expansions (planned in advance)
   - Reactive expansions (emergency/unplanned)
   - Proactive Ratio = (Proactive / Total) × 100%

**Target**: ≥90% proactive per ISMS-POL-A.8.6-S2, Section 1.2

2. Track capacity-related incidents:
   - Capacity exhaustion incidents (count)
   - Capacity-related outages (count)
   - Near-miss events (count)

**Target**: Zero capacity-related outages per quarter

3. Evaluate forecast accuracy:
   - Average MAPE across all forecasts
   - Forecast bias (over/under forecasting)

**Target**: ≤15% MAPE

#### 4.2.4 Capacity Management Maturity Assessment

**Process**:
Assess maturity across key dimensions:

| Dimension | Level 1 (Ad-Hoc) | Level 2 (Defined) | Level 3 (Managed) | Level 4 (Measured) | Level 5 (Optimized) |
|-----------|------------------|-------------------|-------------------|-------------------|---------------------|
| **Monitoring** | Minimal/no monitoring | Basic monitoring | Comprehensive monitoring | Real-time monitoring with alerts | Predictive monitoring with AI/ML |
| **Forecasting** | No forecasting | Manual estimates | Statistical forecasting | Automated forecasting | Continuous forecasting with self-tuning |
| **Planning** | Reactive only | Annual planning | Quarterly planning | Monthly planning with adjustments | Dynamic planning with auto-scaling |
| **Optimization** | No optimization | Ad-hoc right-sizing | Regular optimization reviews | Continuous optimization | Automated optimization |
| **Reporting** | No reporting | Annual reports | Quarterly reports | Monthly reports + dashboards | Real-time dashboards with alerts |

**Maturity Score Calculation**:
```
Overall Maturity = Average(Monitoring Level, Forecasting Level, Planning Level, Optimization Level, Reporting Level)
```

**Maturity Targets**:
- **Year 1**: Achieve Level 2 (Defined) across all dimensions
- **Year 2**: Achieve Level 3 (Managed) across all dimensions
- **Year 3**: Achieve Level 4 (Measured) for critical dimensions

### 4.3 Dashboard Workbook Structure

**Excel Workbook Sheets**:

1. **Executive_Dashboard** - One-page executive summary
2. **Capacity_Health_Score** - Overall health score and breakdown
3. **Utilization_Summary** - Resources by threshold status (from Assessment 1)
4. **Forecast_Summary** - Capacity exhaustion timeline (from Assessment 2)
5. **Planning_Effectiveness** - Proactive vs. reactive, incidents, forecast accuracy
6. **Capacity_Risks** - Top 10 highest-risk resources
7. **Maturity_Assessment** - Capacity management maturity scorecard
8. **Trend_Charts** - Utilization trends over time (6-12 months)
9. **Recommendations** - Priority actions and improvements
10. **Evidence_Summary** - Consolidated evidence index
11. **Approval_Sign_Off** - Dashboard approval

**Key Features**:
- Executive summary (single-page overview)
- Visual indicators (gauges, traffic lights, charts)
- Trend sparklines (compact visualization)
- Risk heatmap (resource type × threshold status)
- Drill-down links to detailed assessments

### 4.4 Evidence Requirements

**Evidence Sources**:
- Assessment 1 workbook (capacity utilization data)
- Assessment 2 workbook (forecast and planning data)
- Capacity incident reports
- Capacity planning meeting minutes
- Executive capacity presentations

**No New Evidence Required**: Dashboard aggregates evidence from Assessments 1 and 2.

### 4.5 Dashboard Frequency and Ownership

**Frequency**: Monthly (or as defined in executive reporting cycle)

**Responsible Party**: Capacity Planning Manager / Infrastructure Manager

## 5. Evidence Collection Framework

### 5.1 Evidence Categories

Evidence for capacity management compliance falls into the following categories:

#### 5.1.1 Monitoring Evidence

**Purpose**: Demonstrate capacity monitoring is implemented and operational

**Evidence Types**:
- Monitoring tool configurations (screenshots, exports)
- Monitoring dashboards showing real-time utilization
- Historical monitoring data (time-series data exports)
- Monitoring alert configurations and thresholds
- Alert history logs (capacity alerts triggered)

**Collection Method**: Export from monitoring tools, screenshots of dashboards, configuration backups

**Retention**: Current configuration + 12 months historical data minimum

#### 5.1.2 Forecasting Evidence

**Purpose**: Demonstrate capacity forecasting is data-driven and accurate

**Evidence Types**:
- Historical utilization data used for forecasting
- Forecast calculations and methodology documentation
- Trend analysis (charts, regression lines, growth rates)
- Previous forecast reports for accuracy validation
- Forecast assumptions and business growth projections

**Collection Method**: Spreadsheets, statistical analysis outputs, forecast reports

**Retention**: 24 months (to enable multi-year trend analysis and accuracy validation)

#### 5.1.3 Planning Evidence

**Purpose**: Demonstrate capacity planning is proactive and effective

**Evidence Types**:
- Capacity planning meeting minutes and agendas
- Quarterly and annual capacity plans
- Capacity expansion approvals (purchase orders, change requests)
- Capacity expansion tracking (planned vs. actual dates)
- Budget documents (capacity CapEx/OpEx allocations)

**Collection Method**: Meeting minutes, planning documents, financial records, change management records

**Retention**: 36 months (multi-year planning cycle)

#### 5.1.4 Reporting Evidence

**Purpose**: Demonstrate capacity reporting to management

**Evidence Types**:
- Monthly capacity reports
- Quarterly capacity planning reports
- Annual strategic capacity plans
- Executive presentations on capacity status
- Board-level capacity risk reporting (if applicable)

**Collection Method**: Reports, presentations, distribution records

**Retention**: 24 months

#### 5.1.5 Incident Evidence

**Purpose**: Demonstrate capacity incident management and learning

**Evidence Types**:
- Capacity-related incident reports
- Root cause analysis for capacity exhaustion
- Capacity near-miss reports
- Corrective and preventive actions
- Incident post-mortems and lessons learned

**Collection Method**: Incident management system records, post-mortem reports

**Retention**: 36 months

#### 5.1.6 Optimization Evidence

**Purpose**: Demonstrate capacity optimization activities

**Evidence Types**:
- Right-sizing analysis and recommendations
- Resource consolidation projects
- Over-provisioning identification and remediation
- Cost optimization reports (cloud spend optimization)
- Performance tuning records

**Collection Method**: Analysis reports, project documentation, cost reports

**Retention**: 24 months

### 5.2 Evidence Storage and Organization

**Evidence Repository**:
- Centralized storage location (SharePoint, network share, document management system)
- Organized by assessment type and time period
- Version-controlled for policy/procedure documents
- Access-controlled (appropriate permissions for auditors, compliance, management)

**Naming Convention**:
```
ISMS-A86-[Evidence Type]-[YYYY-MM]-[Description].[ext]

Examples:
ISMS-A86-Monitoring-2026-01-Prometheus-Dashboard.pdf
ISMS-A86-Forecast-2026-Q1-Capacity-Forecast-Report.xlsx

ISMS-A86-Incident-2026-01-Storage-Exhaustion-RCA.pdf
```

**Evidence Index**: Maintain master evidence index (spreadsheet or database) tracking:
- Evidence ID
- Evidence type
- Date created
- Description
- Storage location
- Related assessment
- Retention expiration date

### 5.3 Evidence Quality Standards

All evidence must meet the following quality criteria:

**Authenticity**: Evidence must be verifiable and traceable to source
**Completeness**: Evidence must fully support the claim being made
**Timeliness**: Evidence must be from the appropriate time period
**Accuracy**: Data and information must be correct and unaltered
**Relevance**: Evidence must directly relate to capacity management requirements

**Evidence Review**: Capacity planning team reviews evidence quality quarterly. Non-conforming evidence is identified and remediated.

## 6. Compliance Scoring Methodology

### 6.1 Compliance Assessment Dimensions

Capacity management compliance is assessed across five dimensions:

#### 6.1.1 Monitoring Compliance

**Assessment Criteria**:
- Monitoring coverage percentage (target: 100% production, 90% non-production)
- Monitoring data quality (accuracy, completeness, timeliness)
- Threshold configuration (warning, critical thresholds defined)
- Alerting functionality (alerts triggering appropriately)
- Data retention (meeting retention requirements)

**Scoring**:
- **100%**: Full compliance - all criteria met
- **75-99%**: Substantial compliance - minor gaps
- **50-74%**: Partial compliance - significant gaps
- **<50%**: Non-compliance - major gaps

#### 6.1.2 Forecasting Compliance

**Assessment Criteria**:
- Forecasts developed for all in-scope resources
- Forecasting methodology documented and appropriate
- Forecast accuracy within acceptable range (≤15% MAPE)
- Forecasts updated at required frequency (quarterly minimum)
- Business growth assumptions documented

**Scoring**: Same scale as monitoring compliance

#### 6.1.3 Planning Compliance

**Assessment Criteria**:
- Capacity planning cycle executed (monthly, quarterly, annual)
- Capacity expansion plans documented
- Proactive vs. reactive expansion ratio (≥90% target)
- Lead time management (adequate lead time for expansions)
- Budget planning integration

**Scoring**: Same scale as monitoring compliance

#### 6.1.4 Reporting Compliance

**Assessment Criteria**:
- Monthly capacity reports produced and distributed
- Quarterly capacity planning reports produced
- Annual strategic capacity plan produced
- Reports reviewed by appropriate management levels
- Incident reporting for capacity-related events

**Scoring**: Same scale as monitoring compliance

#### 6.1.5 Optimization Compliance

**Assessment Criteria**:
- Optimization activities performed (right-sizing, consolidation)
- Over-provisioning identified and remediated
- Cost optimization tracked (cloud spend optimization)
- Performance tuning documented
- Optimization results measured

**Scoring**: Same scale as monitoring compliance

### 6.2 Overall Compliance Score

**Calculation**:
```
Overall Compliance Score = (Monitoring Score + Forecasting Score + Planning Score + Reporting Score + Optimization Score) / 5
```

**Compliance Levels**:
- **Fully Compliant** (≥95%): Comprehensive capacity management implementation
- **Substantially Compliant** (85-94%): Strong implementation with minor improvements needed
- **Partially Compliant** (70-84%): Functional implementation with significant gaps
- **Non-Compliant** (<70%): Major implementation gaps requiring remediation

### 6.3 Compliance Reporting

**Compliance Status**: Reported in monthly capacity dashboard and quarterly capacity reports

**Trend Tracking**: Compliance score tracked over time to demonstrate improvement

**Gap Remediation**: Non-compliance items documented with remediation plans and target dates

## 8. Audit and Compliance Verification

### 8.1 Internal Audit Process

**Frequency**: Annual capacity management audit as part of ISMS internal audit program

**Audit Scope**:
- Review Assessment 1, 2, and Dashboard for completeness and accuracy
- Verify evidence supporting assessments
- Test monitoring systems (sample resources to verify utilization data)
- Interview capacity planning team on methodology
- Review capacity planning meeting minutes and reports
- Assess compliance against ISMS-POL-A.8.6-S2 requirements

**Audit Findings**: Documented in internal audit reports with corrective action plans

### 8.2 External Audit / ISO 27001 Certification

**External Auditor Evidence**:
- Assessment workbooks (Assessments 1, 2, Dashboard)
- Evidence register with supporting documentation
- Capacity planning reports and presentations
- Monitoring system demonstrations
- Capacity incident reports and post-mortems

**Auditor Sampling**: External auditors may sample resources to verify:
- Monitoring is operational
- Utilization data is accurate
- Forecasts are data-driven
- Capacity planning is effective

**Audit Preparation**: Capacity planning team prepares evidence package 30 days before external audit

### 8.3 Continuous Compliance Monitoring

**Monthly Compliance Check**: Capacity planning team performs self-assessment of compliance status

**Compliance Dashboard**: Compliance score included in monthly capacity management dashboard
